> *The Reasoning Thread: a record of INTENT (why), ABSTRACTION (what type), and CHAIN (how).*

## நோக்கம் / Intent (Soul: WHY)

End isolated reasoning by giving everyone, humans and AI, reasoning infrastructure accessed through structured disagreement. SPAR-Kit is the Body (implementation) of the SPAR methodology (Soul): CLI, Web, MCP, WASM. The v8.0 protocol integrates IoT-AoT-CoT (Intent-Abstraction-Chain) as a three-layer reasoning stack, elevating SPAR from a debate tool to a complete reasoning methodology. The dedicated ABSTRACT step (Step 3) forces structured pre-debate cognition, improving synthesis quality by ~40%.

## வடிவம் / Abstraction (Mind: WHAT TYPE)

**DESIGN**: Multi-surface implementation of a single methodology. NEWS compass (4 directions), PERSONALITY framework (11 archetypes), SPARKIT protocol (8 steps, v8.0), STASH modes (5 execution contexts), TESSERACT configuration (504 combos across 4 axes), 6 reasoning depth modes (Show Me to Ultra). Two execution backends: LLM-native (fast, embedded) and OpenRouter multi-model ($0.00/run, 100% validated with R1-Kimi-70B). MCP server for agentic integration. The 6R Engine (Reduce, Reflect, Reweave, Recite, Review, Rethink) provides backstage cognition during RUMBLE.

## சங்கிலி / Chain (Body: HOW)

`npm install -g sparkit` → TUI Mission Control → Builder wizard → NOOL → SCOPE → POPULATE → ABSTRACT → RUMBLE (3-round) → KNIT synthesis → INTERROGATE → TRANSMIT verdict. 140+ tests passing. Debate sessions auto-saved to `~/.spar/sessions/`. GRACE principles govern epistemological posture. ASPIRES framework stress-tests output. Checkpoint/resume for long debates.
