# OSS/IP Boundary Policy

**Version**: 2.0
**License**: MIT (for OSS components)
**Date**: March 2026

---

## 1. Purpose

This repository is an **open standard/toolkit** within the SYNTHAI ecosystem.

This policy defines what is **Open Source (MIT)** vs **Proprietary IP**.

---

## 2. What is OSS (MIT)

- Core specifications, schemas, and protocols
- Reference implementations (SDKs, CLIs)
- Methodology documentation (manifestos, cheatsheets, FAQs)
- Validators, test vectors, conformance suites
- Tutorials and examples
- TESSERACT axis definitions and preset configurations
- Reasoning depth mode specifications
- Abstraction of Thought (AoT) protocol
- NOOL/LOON lifecycle specifications
- 6R Engine specification
- GRACE, ASPIRES, and DOORS frameworks

---

## 3. What is Proprietary IP

- **Books** (published content, narratives, illustrations)
- **Workshop content** (facilitator guides, student workbooks, assessment rubrics)
- **Hosted services** (SaaS, multi-tenant infrastructure)
- **Canon personas** (hand-crafted, culturally grounded persona prompts)
- **Verified datasets** (curated training data, premium benchmarks)
- **Enterprise controls** (SSO, RBAC, compliance, audit logs)
- **GPT/AI system prompts** (coaching cycles, persona architectures)
- **Profiling-to-Reasoning Pipeline** (auto-configuration from user profiles to TESSERACT settings)
- **ARIVAR native packages** (Python backend, ITE integration)
- **Belt-gated progressive disclosure** (SaaS onboarding UX)
- **ZK-Dissent implementation** (cryptographic dissent hashing)
- **SYNTHAI Doctrine** (full organizational philosophy specification)

---

## 4. Trademarks

Even when code/spec is MIT:

- **Trademarks are not granted**
- Names and logos are protected
- You may implement freely but may not claim "official" status without permission

---

## 5. The Boundary Rule of Thumb

> **If it defines the standard or enables local usage → OSS (MIT)**
>
> **If it provides hosted trust, verified content, books, workshops, or enterprise governance → Proprietary IP**

### v2.0 Clarification

The TESSERACT configuration *axes and presets* are OSS (they define the standard). The *mapping logic* that auto-selects configurations from user profiles is proprietary (it powers the hosted service).
