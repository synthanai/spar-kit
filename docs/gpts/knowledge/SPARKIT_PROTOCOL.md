# The SPARKIT Protocol

The 7-step structured methodology for running a SPAR session.

## Overview

| Step | Name | Description |
|------|------|-------------|
| S | Scope | Define the decision, constraints, and context |
| P | Populate | Instantiate personas for each direction |
| A | Announce | Present the challenge to all directions |
| R | Rumble | Let directions argue and clash |
| K | Knit | Synthesize tensions and convergences |
| I | Interrogate | Stress-test the synthesis |
| T | Transmit | Deliver actionable recommendation |

### A â€” Announce
Present the decision equally to all directions. Frame it neutrally:
- Don't lead with your preference
- Include relevant context
- Set the stakes clearly

### R â€” Rumble (THE CORE PHASE)
The dialectic unfolds:
- Each direction argues its authentic position (2-3 paragraphs)
- Arguments should feel like genuine advocacy, not roleplay
- Directions naturally challenge each other
- Tensions surface organically
- No premature consensus or softening

This is where isolated reasoning gets defeated. The goal is productive friction.

### K â€” Knit
Moderator synthesis:
- **Convergence**: Where do ALL directions agree?
- **Key Tensions**: Where do they fundamentally clash?
- **Blind Spots**: What did NOBODY mention?

The synthesis should hold tensions in productive balance, not resolve them artificially.

### I â€” Interrogate
Challenge the synthesis itself:
- What assumption could make this recommendation wrong?
- What voice is still missing?
- What would cause you to reverse this recommendation?
- If this decision fails in 2 years, what was the most likely cause?

This prevents "hallucinated consensus" â€” the synthesis looking more confident than warranted.

### T â€” Transmit
Deliver the final output:
- **Clear Recommendation**: One sentence stating the path forward
- **Specific Next Actions**: 3-5 concrete steps to take
- **Key Risk to Monitor**: The one thing that could invalidate this
- **Confidence Level**: Low/Medium/High with reasoning

## When to Use SPARKIT

- You're going in circles on a decision
- A team agreed too quickly (suspicious consensus)
- High-stakes choice with irreversible consequences
- You need to document the reasoning for stakeholders
- You want to surface your own blind spots

## When NOT to Use SPARKIT

- Decisions that are easily reversible (just try it)
- When data already gives a clear answer
- Trivial choices (don't over-engineer)
- Time-critical emergency responses (act first, SPAR later)
